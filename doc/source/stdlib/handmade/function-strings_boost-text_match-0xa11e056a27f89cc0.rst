Matches ``text`` against a small subset of the SQLite FTS5 query grammar — whitespace-AND terms, ``term*`` prefix match, and ASCII case-folding (UTF-8 multi-byte words preserved as word units). Panics on unsupported syntax (quoted phrases, column filters, ``NEAR(...)``, or the whole-word keywords ``AND`` / ``OR`` / ``NOT`` / ``NEAR``) so callers know to fall back to a SQL backend for the full grammar.
